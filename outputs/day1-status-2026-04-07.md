## Important findings

### sweph in Next.js / Vercel
Without special handling, Next 16 + Turbopack fails to bundle `sweph` with this error:
- `non-ecmascript placeable asset`
- import trace points to `node_modules/sweph/index.mjs`

What fixed local production build:
- `serverExternalPackages: ['sweph']`

Remaining Vercel risk:
- `sweph` is a native Node addon, so it must be executed only in the Node.js runtime, not Edge
- I set the test route to `runtime = 'nodejs'`
- Vercel deployment still needs real preview verification because native module packaging can differ between local and hosted builds
